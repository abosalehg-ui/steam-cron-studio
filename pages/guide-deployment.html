🐳 نشر بـ Docker Compose

  1. ولّد الملفات من Builder

    اضغط Docker Compose في صفحة Builder لتنزيل docker-compose.yml.

  2. أنشئ ملف .env

    .env
    STEAM_WEB_API_KEY=XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX OPENCLAW_TELEGRAM_BOT_TOKEN=1234567890:ABCDEFGxxxxxxxxxxxxxxxxxx # ITAD_API_KEY=your_key # optional, for Task 7

  3. ضع الملفات في مجلد وشغّل

    bash
    mkdir openclaw-steam && cd openclaw-steam # ضع cron_tasks.yaml و docker-compose.yml هنا mkdir state docker compose up -d # تابع السجلات docker compose logs -f

  4. تحقق أن الحاوية تعمل

    bash
    docker compose ps docker compose exec openclaw-steam openclaw status
✅
مميزات Docker: Chromium مدمج، إعادة تشغيل تلقائية، عزل كامل، سهولة التحديث.

⚙️ نشر بـ Systemd

  1. ولّد ملفات Systemd من Builder

    اضغط Systemd Service — ستحصل على 3 ملفات: service، timer، وinstall.sh.

  2. ارفع الملفات للسيرفر

    bash
    scp cron_tasks.yaml openclaw-steam.service openclaw-steam.timer install.sh user@server:~/

  3. شغّل سكريبت التثبيت

    bash
    chmod +x install.sh sudo ./install.sh # حرّر ملف .env بعدها sudo nano /opt/openclaw/.env

  4. تحقق من الحالة

    bash
    systemctl status openclaw-steam journalctl -u openclaw-steam -f --output=cat
ℹ️
Systemd يديرها كـ daemon — يبدأ تلقائياً عند restart السيرفر ويعيد المحاولة عند الأخطاء.

🐍 Python + crontab

  1. ثبّت openclaw والمتطلبات

    bash
    python3 -m venv ~/openclaw-env source ~/openclaw-env/bin/activate pip install openclaw playwright playwright install chromium

  2. ضع متغيرات البيئة

    bash
    cat >> ~/.bashrc << 'EOF' export STEAM_WEB_API_KEY="your_key" export OPENCLAW_TELEGRAM_BOT_TOKEN="your_token" EOF source ~/.bashrc

  3. شغّل openclaw بـ crontab

    bash
    crontab -e # أضف هذا السطر (openclaw يقرأ الجدول من الـ YAML): @reboot /bin/bash -c 'source ~/.bashrc && ~/openclaw-env/bin/openclaw run ~/cron_tasks.yaml' &

  4. شغّل يدوياً للاختبار

    bash
    source ~/openclaw-env/bin/activate openclaw run cron_tasks.yaml --task steam_free_deals_daily --now

🔧 حل المشاكل الشائعة

Chromium / Playwright لا يعمل

مشكلة شائعة في السيرفرات بدون GUI.

bash
# تثبيت dependencies للـ headless Chromium sudo apt-get install -y libnss3 libatk-bridge2.0-0 libdrm2 libxcomposite1 \ libxdamage1 libxfixes3 libxrandr2 libgbm1 libpango-1.0-0 libcairo2 # أو عبر Playwright مباشرةً playwright install-deps chromium

Telegram: "Bad Request: can't parse entities"

هذا خطأ الـ parse_mode. الـ YAML المولَّد يتضمن parse_mode: HTML في كل مهمة — تأكد أنك تستخدم YAML المولَّد من Builder v3.0.

Steam API: 401 / 403

تأكد أن STEAM_WEB_API_KEY صحيح وأن حسابك غير مقيّد. أعد توليد المفتاح إذا لزم من steamcommunity.com/dev/apikey

المهمة 3 أو 8 لا تُرسل صورة

تحقق من ./state/render_errors.log — المهمة تفشل إلى إرسال نص في حال فشل Playwright.

bash
cat ./state/render_errors.log
